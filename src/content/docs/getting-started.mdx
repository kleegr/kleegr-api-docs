Kleegr's API platform is powered by HighLevel APIs. This guide walks you through registering an OAuth app, getting an access token, and calling your first endpoint.

## Prerequisites

- A Kleegr CRM account — [crm.kleegr.com](https://crm.kleegr.com)
- A developer account at the HighLevel Marketplace (required for OAuth app registration)

## Quick start

<Steps>

1. **Register an OAuth app**

   Go to [marketplace.gohighlevel.com](https://marketplace.gohighlevel.com), sign in, navigate to **My Apps** → **Create App**. Set your redirect URI and note your `client_id` and `client_secret`.

2. **Build the authorization URL**

   ```
   https://marketplace.gohighlevel.com/oauth/chooselocation?
   response_type=code&redirect_uri=https://yourapp.com/callback&
   client_id=YOUR_CLIENT_ID&scope=contacts.readonly contacts.write
   ```

3. **Exchange the code for a token**

   ```bash
   curl -X POST https://services.leadconnectorhq.com/oauth/token \
     -H "Content-Type: application/x-www-form-urlencoded" \
     -d "grant_type=authorization_code" \
     -d "client_id=YOUR_CLIENT_ID" \
     -d "client_secret=YOUR_CLIENT_SECRET" \
     -d "code=AUTH_CODE" \
     -d "redirect_uri=https://yourapp.com/callback"
   ```

4. **Make an API call**

   ```bash
   curl https://services.leadconnectorhq.com/contacts/search \
     -H "Authorization: Bearer ACCESS_TOKEN" \
     -H "Version: 2021-07-28"
   ```

</Steps>

## Rate limits

| Limit | Value |
|-------|-------|
| Burst | 100 requests / 10 seconds per app per location |
| Daily | 200,000 requests / day per app per location |

Headers: `X-RateLimit-Limit-Daily`, `X-RateLimit-Daily-Remaining`, `X-RateLimit-Remaining`

## Token lifecycle

- **Access tokens**: valid 24 hours — refresh with the refresh token
- **Refresh tokens**: valid 1 year, reset on each use
