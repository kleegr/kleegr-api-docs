Kleegr uses the **OAuth 2.0 Authorization Code Grant** flow. Your app requests permission to access a CRM location on a user's behalf, receives an authorization code, and exchanges it for access and refresh tokens.

## Flow summary

1. Send the user to the **Authorization URL** with your `client_id` and `scopes`
2. The user selects their Kleegr CRM location and grants permission
3. They are redirected to your `redirect_uri` with a `code`
4. Your server exchanges `code` for `access_token` + `refresh_token`
5. Use `access_token` in `Authorization: Bearer` header on every API call

## Base URLs

| Purpose | URL |
|---------|-----|
| Standard authorization | `https://marketplace.gohighlevel.com/oauth/chooselocation` |
| White-label authorization | `https://marketplace.leadconnectorhq.com/oauth/chooselocation` |
| Token exchange | `https://services.leadconnectorhq.com/oauth/token` |
| All API calls | `https://services.leadconnectorhq.com` |
