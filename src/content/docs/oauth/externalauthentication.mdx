External Authentication allows Kleegr Marketplace apps to authenticate with third-party services using OAuth 2.0 or API Key flows.

## What is External Authentication?

When your app needs to connect to a third-party service on behalf of your users, External Authentication handles the OAuth flow and securely stores the credentials. Users authenticate once during app installation and the tokens are managed automatically.

## Supported authentication types

| Type | Description |
|------|-------------|
| OAuth 2.0 | Standard OAuth authorization code flow |
| API Key | Simple API key authentication |
| Username/Password | Basic credential authentication |

## OAuth 2.0 Configuration

Configure the following in your Marketplace app settings:

### Authorization URL

The URL where users are redirected to authorize your app with the third-party service.

### Access and Refresh Token Request

Configure the token request settings using these standard parameters:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `client_id` | `{{externalApp.clientId}}` | Unique identifier issued by the third-party application |
| `client_secret` | `{{externalApp.clientSecret}}` | Confidential key used to authenticate your application |
| `scope` | `{{externalApp.scope}}` | Space-separated list of permissions required |
| `state` | `{{bundle.state}}` | Security token to prevent CSRF attacks |
| `redirect_uri` | `{{bundle.redirectUrl}}` | Callback URL registered in the third-party app |

### Token Exchange Parameters

| Parameter | Value | Description |
|-----------|-------|-------------|
| `code` | `{{bundle.code}}` | Authorization code received after user approval |
| `access_token` | `{{bundle.accessToken}}` | Token used to authenticate API requests |
| `refresh_token` | `{{bundle.refreshToken}}` | Long-lived token to obtain new access tokens |

## Example token response

```json
{
  "access_token": "your_access_token",
  "refresh_token": "your_refresh_token",
  "expires_in": 3600
}
```

## Auto Refresh Token

Enable Auto Refresh to automatically refresh expired access tokens without requiring user re-authentication. When enabled, Kleegr uses the refresh token to get a new access token before it expires.

## Accessing user-provided data

If your authentication requires user input (API Key, Username, Password), access the values using:

```javascript
userData.apiKey      // if field key is 'apiKey'
userData.username    // if field key is 'username'
userData.password    // if field key is 'password'
```

The `userData` object contains all values entered by the user during installation.

## User context request body

When verifying user context, the request body includes location information:

```json
{
  "companyId": "123",
  "locationId": "[selected location IDs]",
  "username": "user1",
  "password": "password123",
  "approveAllLocations": false,
  "excludedLocations": null
}
```

## Important notes

- External Authentication is backward compatible. Existing apps continue working without changes.
- If you update external auth settings, existing users need to reinstall the app for changes to take effect.
- The `state` parameter must be returned unchanged in the callback response or the request will be rejected.
